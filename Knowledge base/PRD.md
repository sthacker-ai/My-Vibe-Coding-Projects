# Product Requirements Document — KnowledgeBase

**Version:** 3.0 (post Phase 1–3, run history, token tracking, courseware UX)
**Last Updated:** May 2026

---

## 1. Product Vision

KnowledgeBase is a **local-first personal learning system** that turns your X/Twitter liked posts, YouTube videos, and articles into a structured, searchable knowledge base — with AI-generated courseware, a wiki, and a knowledge graph.

Everything runs on your own machine. No cloud dependency for content storage. PostgreSQL is used for learning progress tracking and pipeline observability. The project started small and is evolving into a full learning management system — architecture is intentionally kept hybrid (files + PostgreSQL) so content remains git-trackable and Obsidian-compatible.

## 3. Architecture

### 3.1 Data Flow

```
X/Twitter Likes
    → import-x-likes.js (Playwright)
    → data/raw/{tweet_id}.json

YouTube / X Video
    → import-youtube.js / extract-transcripts.js
    → yt-dlp download
    → Groq API transcription (whisper-large-v3-turbo, free tier) — ~10 sec per file
      OR faster-whisper local fallback (CPU, ~20-30 min per file)
    → data/transcripts/{tweet_id}.txt

data/raw/*.json
    → import-x-source.js (Playwright scraper)
    → extracted content saved to raw JSON

Classified sources
    → classify-source.js + OpenRouter AI
    → topic_slug, topic_label assigned

Classified sources + transcripts
    → compile-course.js + OpenRouter AI
    → content/courses/{topic_slug}/course-NNN.md

Courses
    → update-topic-summary.js + OpenRouter AI
    → content/wiki/{topic_slug}.md

All content
    → build-graph.js
    → data/indexes/graph.json

Next.js app reads from:
    - content/courses/**/*.md
    - content/wiki/*.md
    - data/indexes/
    - PostgreSQL (learning_progress, reading_sessions, daily_streaks)
```

### 3.2 AI Provider Chain

1. **OpenRouter primary:** `nvidia/nemotron-3-super-120b-a12b:free`
2. **OpenRouter fallback:** `google/gemma-4-31b-it:free`
3. **Ollama local fallback:** `gemma4:e2b` at localhost:11434
4. **Groq (transcription only):** `whisper-large-v3-turbo` — free tier at console.groq.com

### 3.3 Storage Strategy (Hybrid)

| Data | Storage | Reason |
|---|---|---|
| Courses, wiki, sources | File system (`.md`, `.json`) | Static at build time, version-controllable, Obsidian-compatible |
| Learning progress | PostgreSQL | Mutable, queryable, survives rebuilds |
| Reading sessions | PostgreSQL | Time-series, needs aggregation |
| Daily streaks | PostgreSQL | Incremental updates |
| Pipeline run history | `data/runs/` JSON + PostgreSQL (planned) | Observability; JSON is immediate, DB enables querying |
| AI token usage | `data/token-usage.json` + PostgreSQL (planned) | Track cost per model; JSON is immediate, DB enables analytics |
| AI prompts | `data/prompts.json` | User-editable, hot-reloaded |
| Graph data | `data/indexes/graph.json` | Rebuilt on demand |

## 4. Pipeline Steps

| Step | Script | Input | Output |
|---|---|---|---|
| 1. Import likes | `import-x-likes.js` | X/Twitter account | `data/raw/*.json` |
| 2. Extract sources | `import-x-source.js` | raw JSON | extracted text in raw JSON |
| 3. Classify | `classify-source.js` | extracted JSON | `topic_slug` added |
| 4. Compile courses | `compile-course.js` | classified JSON + transcript | `content/courses/` |
| 5. Summarize topics | `update-topic-summary.js` | courses | `content/wiki/` |
| 6. Build graph | `build-graph.js` | all content + indexes | `data/indexes/graph.json` |
| 7. Export Obsidian | `export-obsidian.js` | all content | `data/obsidian-vault/` |
| 8. Daily schedule | `scheduled-daily.js` | cron trigger | runs steps 1–6 |

## 6. Admin Console Features

- **Task Terminal** — live output from any pipeline step
- **Import Sources** — import YouTube/X video URLs directly
- **Pipeline Controls** — run any step or full pipeline with one click
- **AI Prompts Editor** — edit system prompts and instructions for all 3 AI tasks; saved to `data/prompts.json`; hot-reloaded on next run
- **Recent Activity** — log of recent pipeline task runs with status

## 7. Learning Tracker (PostgreSQL)

### Tables

```sql
learning_progress  (id, course_id, topic_slug, status, started_at, completed_at, time_spent_sec)
reading_sessions   (id, course_id, session_date, duration_sec)
daily_streaks      (streak_date, minutes_spent, sessions_count)
```

### API Routes

| Method | Route | Description |
|---|---|---|
| POST | `/api/learn/progress` | Upsert course progress + log reading session |
| GET | `/api/learn/stats` | Aggregated stats: streaks, completion, total time |

---

## 8. Knowledge Graph

- Built by `build-graph.js` → `data/indexes/graph.json`
- Node types: `topic` (purple), `source` (blue), `course` (amber)
- Edge types: `belongs_to`, `wiki_link`, `tagged`, `cites`
- Rendered via D3 force simulation at `/graph`
- Click topic node → navigates to wiki page
- Click source node → opens tweet URL in new tab
- Supports drag, zoom, pan

---

## 9. Obsidian Export

- Script: `scripts/export-obsidian.js`
- Output: `data/obsidian-vault/` (configurable via `OBSIDIAN_VAULT_DIR` in `.env`)
- Structure: `Topics/`, `Courses/[Topic]/`, `Sources/`, `Index.md`
- All cross-references converted to `[[wikilinks]]`
- Frontmatter added to every file with tags, topic, source metadata
