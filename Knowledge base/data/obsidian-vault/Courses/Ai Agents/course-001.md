# Automating Agent Workflows with Orchestration

## Overview
This course covers how to utilize advanced agent orchestration to automate complex task decomposition and specialized assignment within an Agent Kanban system. It focuses on leveraging a single input prompt to trigger intelligent, multi-step execution.

## Key Concepts
- AI Agents
- Agent Kanban
- Orchestrator Agent
- Triage
- Task Decomposition
- Agent Profiles

## Chapter 1: Agent Orchestration
The Orchestrator agent serves as the central brain for the system, taking an initial prompt and managing the entire workflow. It is responsible for understanding the request and initiating the subsequent automation steps.

## Chapter 2: Intelligent Decomposition and Assignment
Once the prompt is received, the orchestrator decomposes the request into all necessary subtasks. It then automatically assigns these subtasks to specific agent profiles, ensuring that the required specialization is matched for optimal execution.

## Key Takeaways
- Automation is achieved by feeding a single prompt into the triage system, allowing the orchestrator to manage complexity.
- The orchestrator's primary role is task decomposition and intelligent agent profile assignment.
- Specialization is key: Agents are automatically assigned based on the specific skills required for a subtask.
- The system transforms a high-level request into a complete, specialized execution plan.
